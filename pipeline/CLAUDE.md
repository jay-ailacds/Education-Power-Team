# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Build & Run Commands

```bash
# Run any pipeline step (no build step needed — uses tsx for direct TS execution)
npx tsx src/cli.ts <command> projects/<config>.yaml [--force] [--verbose] [--dry-run]

# Individual step commands
npx tsx src/cli.ts render projects/education-power-team.yaml    # Full pipeline
npx tsx src/cli.ts research projects/education-power-team.yaml  # Exa.ai research only
npx tsx src/cli.ts enhance projects/education-power-team.yaml   # Apply enhancements
npx tsx src/cli.ts tts projects/education-power-team.yaml       # Generate voiceovers
npx tsx src/cli.ts music projects/education-power-team.yaml     # Generate music
npx tsx src/cli.ts video projects/education-power-team.yaml     # Generate video clips
npx tsx src/cli.ts mix projects/education-power-team.yaml       # Mix audio
npx tsx src/cli.ts compose projects/education-power-team.yaml   # Final video composition
npx tsx src/cli.ts status projects/education-power-team.yaml    # Show pipeline state
npx tsx src/cli.ts resume projects/education-power-team.yaml    # Resume from last step

# Reset & regenerate commands
npx tsx src/cli.ts reset projects/education-power-team.yaml tts       # Reset tts + all downstream
npx tsx src/cli.ts reset projects/education-power-team.yaml slides    # Reset slides + compose
npx tsx src/cli.ts reset projects/education-power-team.yaml --all     # Reset everything
npx tsx src/cli.ts regen projects/education-power-team.yaml --scene scene5a-vardhman --step tts    # Regen one scene's voiceover
npx tsx src/cli.ts regen projects/education-power-team.yaml --scene scene5a-vardhman --step video  # Regen one scene's video clip

# Type checking (no compiled output — project uses tsx runtime)
npx tsc --noEmit

# npm script shortcuts
npm run render -- projects/education-power-team.yaml
npm run status -- projects/education-power-team.yaml
```

There are no test or lint commands configured.

## Architecture

### Pipeline Flow

An 8-step sequential video production pipeline with idempotent state management:

```
research → enhance → tts → music → video → mix_audio → compose → verify
```

Each step is tracked in `.pipeline-state.json` (in the output directory). Steps can be individually rerun with `--force` or the pipeline resumes from the last completed step automatically.

Step dependency graph (resetting a step auto-resets all downstream):
```
research → enhance → tts ─┬→ slides → compose → verify
                           ├→ mix_audio ──↗
              music ───────┘
              video ──→ slides
```

### Module Layout

- **`src/cli.ts`** — Commander.js CLI entry point. All commands route through `pipeline.ts`.
- **`src/core/pipeline.ts`** — Main orchestrator. Runs steps in order, manages state transitions, loads cached results between steps. This is the "glue" file.
- **`src/core/state.ts`** — `PipelineStateManager` persists step status/outputs to JSON. Backfills new steps automatically when step order changes. `resetWithCascade()` uses the `DOWNSTREAM` dependency map to auto-reset dependent steps.
- **`src/types.ts`** — All shared interfaces. `StepName` union type must match `STEP_ORDER` in `state.ts`.
- **`src/config/schema.ts`** — Zod schemas for YAML project configs. Adding a new config field requires updating both the Zod schema and the `ProjectConfig` type.
- **`src/api/kie-client.ts`** — Kie.ai HTTP client. All AI generation (TTS, music, video) goes through this. Tasks are async: create → poll → download. Polling uses exponential backoff (3s initial, 1.5x multiplier, 30s cap).
- **`src/api/research.ts`** — Exa.ai search client. Outputs `research-report.json` to the output directory. Non-blocking (pipeline continues if research fails).
- **`src/api/enhance.ts`** — Script enhancement. Enhanced scripts are pre-generated by Claude in-session and saved to `enhanced-scripts.json`. The pipeline loads and applies them — it does NOT call any LLM API. Only use Kie.ai for TTS/music/video generation.
- **`src/api/tts.ts`** — ElevenLabs TTS via Kie.ai. Flattens scenes+segments into individual jobs, runs in parallel.
- **`src/api/video-gen.ts`** — Veo/Runway video via Kie.ai. Supports image-to-video (using images from `assets/`). Runs sequentially to avoid rate limits.
- **`src/media/ffmpeg.ts`** — FFmpeg subprocess wrappers (concat, mix, mux, trim, probe). Requires FFmpeg 7+ installed.
- **`src/media/audio-mixer.ts`** — Concatenates TTS clips with silence gaps, trims music, mixes at configured dB levels.
- **`src/media/video-composer.ts`** — Final mux of video + mixed audio with `-movflags +faststart`.

### Key Design Decisions

- **No LLM API for enhancements**: The `enhance` step loads pre-generated `enhanced-scripts.json` from disk. Enhancements are created by Claude in the active session, not by calling Kie.ai's chat API. This avoids rate limits and cost.
- **Kie.ai returns HTTP 200 for errors**: The API sometimes returns `{"code":500,"msg":"..."}` with a 200 status. Always check the response body, not just `res.ok`.
- **Config-driven**: All project settings live in YAML files under `projects/`. Adding a new video project means creating a new YAML config.
- **State backfill**: When new steps are added to `STEP_ORDER`, the state manager automatically adds them as `pending` to existing state files.
- **Scenes can have segments**: A scene can either have a `script` directly or contain `segments[]` with individual scripts. Both `tts.ts` and `enhance.ts` flatten this structure.

## Environment

- **Runtime**: Node.js with `tsx` (TypeScript Execute) — no build/compile step
- **Module system**: ESM (`"type": "module"`) with `.ts` extension imports
- **TypeScript**: Strict mode, ES2022 target, bundler module resolution
- **Required**: `KIE_API_KEY` in `.env` (for TTS, music, video generation)
- **Optional**: `EXA_API_KEY` in `.env` (for research step)
- **Required external**: FFmpeg 7+ on PATH
